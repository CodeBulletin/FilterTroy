Overview

Filter Troy is designed to allow users to create and share their custom image filters using Python code. To facilitate this, we accept Python code submissions with a specific entry point function. This function will process an image using user-defined parameters and return the processed image. Below is the required structure and explanation of the code input for creating these filters.

Function Signature
def filter(image: np.ndarray, variables: Dict[str, str]) -> np.ndarray:
Parameters
  • Image(np.ndarray):
    • A numpy array representing the image to be processed. The image will typically be in the form of a multi-dimensional array where each element represents pixel values.
  • Variables(Dict[str, str]):
    • A dictionary containing filter parameters. Each key-value pair represents a variable name and its value. Although users may enter variables as integers, floats, strings, etc., all variables are converted to strings before being passed to this function. It is the responsibility of the function to appropriately parse these string values back into their intended data types if necessary.
Return Value

The function must return a numpy array (np.ndarray) that represents the processed image. This array should be of the same shape and data type as the input image array unless intentionally altered by the filter logic.

Example Usage

Hereâ€™s an example of a simple filter function that adjusts the brightness of an image. The brightness level is passed as a variable: 

from typing import Dict

def filter(image: np.ndarray, variables: Dict[str, str]) -> np.ndarray:
    # Convert the brightness variable from string to integer
    brightness = int(variables["brightness"])

    # Clip the values to ensure they fall within the byte range
    return np.clip(image + brightness, 0, 255)

In this example, the variables dictionary might look like this: 

 {"brightness": "50"}
Notes on Data Type Conversion

Since all variables are passed as strings regardless of how they are input by the user, it is important for your filter function to handle type conversion safely and effectively. Here are some common scenarios:

  • Converting to integer: 
    int(variables["var_name"])
  • Converting to float: 
    float(variables["var_name"])
  • Using the value as a string: 
    variables["var_name"]

Always ensure you handle potential errors in conversion (e.g., ValueError when converting an invalid string to a number).

Conclusion

Your custom filter function must adhere to the specified signature and handle the conversion of variables from strings as needed. By doing so, you can implement a wide range of effects and transformations on the images, enhancing the creativity and functionality offered by Filter Troy.
