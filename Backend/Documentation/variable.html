Overview

Filter Troy allows filter creators to define variables that users can adjust to modify how the filter processes images. These variables are defined using a JSON structure, enabling a flexible and user-friendly way to specify input controls such as sliders, text fields, selection menus, and checkboxes. Below, we provide details on how to define each type of input.

Variable Definition Structure

Each variable in the filter configuration is defined as a JSON object with properties that describe the type of input and its default settings. Here is a breakdown of the input types and their respective properties:

  1. Slider
    • type: “slider"
    • min: The minimum value of the slider.
    • max: The maximum value of the slider.
    • step:The smallest increment by which the slider can adjust the value.
    • value: The default value set when the filter is first loaded.
  2. Text
    • type: “text”
    • dtype: Specifies whether the text input is treated as plain “text” or a “number”.
    • value: The default text or number present in the text field.
  3. Numbers
    • type: “text” (with "dtype": "number")
    • value: The default numerical value.
  4. Color
    • type: “color”
    • value: The default color value, represented as a hex code (e.g., “#FF0000”).
  5. Select
    • type: “select”
    • options: An array of objects where each object has a "value" key for the data value and a "label" key for the display label.
    • value: The default selected value.
  6. Checkbox
    • type: “checkbox”
    • value: A boolean that represents whether the checkbox is checked (true) or unchecked (false).
Example JSON Definition
{
    "brightness": {
        "type": "slider",
        "min": 0,
        "max": 100,
        "step": 1,
        "value": 50
    },
    "caption": {
        "type": "text",
        "dtype": "text",
        "value": "Enter caption"
    },
    "scale": {
        "type": "text",
        "dtype": "number",
        "value": 1
    },
    "background_color": {
        "type": "color",
        "value": "#FFFFFF"
    },
    "effect_type": {
        "type": "select",
        "options": [
            {
                "value": "blur", 
                "label": "Blur"
            },
            {
                "value": "sharpen", 
                "label": "Sharpen"
            }
        ],
        "value": "blur"
    },
    "invert_colors": {
        "type": "checkbox",
        "value": false
    }
}
Usage Notes
  • Ensure that the default values provided in the JSON configuration are valid and meaningful for the filter’s operation.
  • For select inputs, ensure that the value specified in the default setting matches one of the options’ values.
  • Handle parsing and error checking in your filter function to accommodate the string-to-type conversion for inputs that are not inherently string-based (e.g., numbers, booleans).
